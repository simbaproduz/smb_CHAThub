SIMBA | youtube-live-adapter-v1

Guia do YouTube

O YouTube agora tem um caminho principal e um fallback avancado. O principal e compatibilidade real por live publica. O oficial fica guardado para os casos em que a compatibilidade nao bastar.

Regra de produto Credenciais do YouTube pertencem ao usuario Elas ficam no arquivo local privado do monitor e nao entram no repositório nem no pacote publicado.

Modo Principal

Compatibilidade real sem credencial obrigatoria
  1. Abra `Configuracao`.
  2. Cole a live URL, channel URL, `@handle` ou channel ID no bloco `Comecar rapido`.
  3. O monitor resolve o alvo publico e tenta abrir o chat real por compatibilidade.
  4. Se a live estiver offline, restrita ou sem chat, a tela mostra isso com honestidade.

Fallback Oficial

Quando a compatibilidade nao bastar
  1. Salve primeiro o link da live, `@handle` ou channel ID.
  2. Abra `Fallback oficial` no card do YouTube.
  3. Informe uma `API key` do YouTube Data API v3. `Access token` fica como alternativa local.
  4. Salve o fallback oficial. O runtime tenta esse caminho so depois de falhar ou nao sustentar a compatibilidade.

Como conseguir a API Key

Fluxo minimo para usuario novo
  1. Abra a pagina de credenciais do Google Cloud.
  2. Crie ou selecione um projeto.
  3. Ative `YouTube Data API v3` para esse projeto.
  4. Clique em `Create credentials` > `API key`.
  5. Restrinja a chave para `YouTube Data API v3` antes de usar em producao.

O que fica local

Sem segredo no repositório
Base publica

`runtime/monitor-config.json` fica limpa e pronta para publicar.

Override privado

`runtime/monitor-config.local.json` guarda API key, access token e estado local do usuario.

Limite honesto

O que o app nao finge

A compatibilidade tenta a live publica sem segredo do usuario, mas ela continua sujeita a lives offline, privadas, members-only, age-restricted ou sem chat ativo. Nesses casos o app nao inventa sucesso.
